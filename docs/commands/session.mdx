---
title: session
description: Save and restore browser sessions.
---

```bash
mosey session save <name>
mosey session load <name>
mosey session list
```

Persist and restore browser state (cookies, storage) across conversations.

## Commands

### save

```bash
$ mosey session save stripe
{"saved":true,"name":"stripe","domains":["stripe.com","dashboard.stripe.com"]}
```

Saves all cookies and storage for the current browser context to `~/.mosey/sessions/<name>/`.

### load

```bash
$ mosey session load stripe
{"loaded":true,"name":"stripe"}
```

Restores cookies and storage from a saved session. Navigate after loading to use the restored auth state.

### list

```bash
$ mosey session list
[
  {"name":"stripe","domains":["stripe.com"],"lastUsed":"2026-04-10T14:30:00Z"},
  {"name":"github","domains":["github.com"],"lastUsed":"2026-04-08T09:15:00Z"}
]
```

Lists all saved sessions with their associated domains and last-used timestamps.

## Examples

```bash
# Save after logging in
$ mosey navigate https://stripe.com/login
$ mosey do r5 --value "user@example.com"
$ mosey do r6 --value "password123"
$ mosey do r7  # click login
$ mosey session save stripe

# Restore in a future conversation
$ mosey session load stripe
$ mosey navigate https://dashboard.stripe.com
# Authenticated automatically via saved cookies

# Check if a session exists for a domain
$ mosey session list | jq '.[] | select(.domains[] | contains("stripe"))'
```

## Storage

```
~/.mosey/sessions/
├── stripe/
│   ├── cookies.json    # Browser cookies
│   ├── storage.json    # localStorage + sessionStorage
│   └── meta.json       # Domains, timestamps
└── github/
    └── ...
```

## Notes

- Sessions are stored unencrypted on disk — treat them like browser profiles
- Cookies may expire between saves and loads; verify auth state after loading
- Loading a session replaces the current browser context's cookies/storage
